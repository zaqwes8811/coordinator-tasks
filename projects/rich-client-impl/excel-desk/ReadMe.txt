Самые важные :
src - файлы с программками
statistic - файлы для обработки; описания задач(ХP-истории), проблем
dist - то, что реально работает(сделано, протещено, поюзано) - дистрибутивы
tests - тесты если таковые будут нужны

Прочие :
git-summary - премеры использования git на резные случаи жизни
utf-8-template.txt - !!важно!! любой текстовой файл создавать копирую этот
  это из-за проблем с русским языком
bugzzz.txt - глюки, проблемы, замечания - называетя багтрекер

pull and push.bat - !!важно!! лучше запускать так. клацнуть дважды bat.bat
  откроется окно коммандной строки. В ней вписать pull or push. Можно 
  просто их два раза кликнуть, но если из комм. строки, то можно увидеть, 
  если что-то пойдет не так, что пошло не так - например конфликты слияния
